# Meeting Rooms ↔ Floor Kiosk — Room Status & Booking

## Summary
The Meeting Rooms Kiosk is a native tablet app (Android/iOS/FireOS) mounted outside each
meeting room. It is powered by Meeting Rooms data but shares device infrastructure (MDM, pairing,
hardware) with the wider Floor Kiosk module. Maintenance banners also flow from Meeting Rooms
to the kiosk surface.

## Modules Involved
- [[modules/meeting-rooms]] — data owner (room status, bookings, maintenance periods)
- [[modules/floor-kiosk]] — device infrastructure owner (MDM, pairing, tablet hardware)

## How They Connect
The Meeting Rooms Kiosk is paired to a specific room using a pairing code generated by Meeting Rooms.
Once paired, the kiosk receives:
- Room status in real-time (available/upcoming/booked)
- Today's booking schedule
- Maintenance period banners (from [[entities/maintenance-period]])

The kiosk can also **write** back to the Meeting Rooms backend:
- Ad-hoc bookings (Book Now / Book for Later)
- Check-in events (PIN or QR)
- Meeting extension or early release
- Cancellation (PIN-verified, email OTP)

## Shared Data Flows

```
[meeting-rooms backend]
   Room pairing code generation
   Real-time status feed (available/occupied/maintenance)
   Booking calendar (today only on kiosk)
             │
             ▼
[floor-kiosk MDM infrastructure]
   Device enrolled, single-app mode, supervised
   Pairing code + admin email → room association stored
             │
             ▼
[Meeting Rooms Kiosk UI (native tablet app)]
   Displays: room name, status colour, upcoming bookings, maintenance banner
   Accepts: Book Now, Book Later, Check-in (PIN/QR), Extend, Cancel, Release
             │
             ▼
[meeting-rooms backend writes]
   New booking created or existing booking mutated
```

## Colour Coding (Room Status)
| Colour | Meaning |
|--------|---------|
| Green | Available |
| Orange | Meeting starting within 30 minutes |
| Red | In use / Booked |
| Yellow | Under maintenance ([[entities/maintenance-period]]) |
| (texture overlay) | Colour-blind accessible variant |

## Potential Conflicts
- **MDM policy** determines whether the kiosk app can receive OTA updates — delay in MDM propagation vs. backend API version mismatch is a real risk when Meeting Rooms deploys new kiosk features.
- `CANCEL_EVENT_PIN_VERIFICATION_ENABLE` and `MEETING_EMAIL_OTP_TO_AUTHENTICATE` config flags must be consistent — a mismatch causes confusing auth prompts on kiosk.
