PARKING MANAGEMENT PRD

Problem Statement:
Prior to the pandemic, parking spaces in organizations were often reserved to a few employees. With the increased trend in hybrid working, these reserved parking spaces are often underutilized. These spaces are now opened for all employees on a first come, first serve basis. Most companies have limited parking spaces available for utilization by their employees as compared to desk availability which creates ambiguity leading to inconvenience to the employees planning to work from the office.
Further, in larger tech parks, employees/visitors of a particular office are often clueless on where the parking premises are located, resulting in in-efficient utilization of the parking spaces. 

Objective:
Parking management solution aims to provide a real time status and visibility of the parking space. Employees can book parking slots in advance. Unclaimed parking slots will be available again for booking which will help companies better manage their parking spaces and ensure efficient utilization of the resources. 

Who’s it for?
	•	Companies with limited spaces available for parking
	•	Owners that lend parking spaces to various companies and individuals

Why build it?
	•	Sales leads are generated requesting for this feature. It would help us sell better
	•	Helps us enrich our product suite

Grouping the parking slot and assigning it to certain employees. 

Problem: As an Admin, I want to assign a certain number of parking slots for certain employees such that only those certain employees can book it. 

There are 2 ways of assigning parking:
	•	Assign to a specific employee or Teams/Org/BL
	•	Eg. Assigning a slot to Ujjwal or Assigning 10 slots to the marketing team
Solution: 
	•	Admin should be allowed to assign parking slots as one of the following:
	•	Hotslot: Multiple slots open for anyone to book
	•	Employee: Single slot open to be booked only by a single assigned employee
	•	Team: Multiple slots open to be booked only by members of assigned team 
	•	Blocked: Multiple slots Not open for booking by anyone
	•	Unallocated: Multiple slots that don’t have any of the above assignments. These slots are not open for booking by anyone
	•	By Default all the slots are Hotslots
	•	When employees opts for parking: 
	•	Dedicated slots should be sequentially assigned to them.
	•	Rules: 
	•	Employees associated with a dedicated “Employee” slot should get that slot assigned to them. Auto populate the slot in the WFO booking form.
	•	Employees belonging to a team associated with the “Teams'' slot should be able to select a slot from within those groups of slots assigned to their teams
	•	Employees with no dedicated “Employee” slots or not associated with any team (which has dedicated slots) should be able to assign from hotslots. 
	•	If the slots are over for a particular team, the employees should be able to select a slot from hotslot
	•	No employee should be able to book parking slots which are unallocated or blocked.  

	•	Assignment based on Tags
	•	Eg: I want 10 slots reserved only for PWD (person with disability) employees, 15 slots for pregnant women etc.
Solution: 
	•	Admin should be able to create a tag type - tag values pair and assign it to a certain parking slots
	•	Eg: Tag Type:IsPWD? Tag Value: Yes/No
	•	Admin should be able to create and assign tags to employees
	•	Eg; Is tag Type: PWDEligible? Tag Value: Yes/No
	•	Rule: The rule engine should allow to map these tags and assign properties
	•	Eg: slots with IsPWD - Yes should be able to be booked by employees with PWDEligible - Yes
	•	Rule: There should be a property assigned to every tag type - tag value pair that defines the restrictions level.
	•	Users are allowed to only book from those tagged slots
	•	Users are allowed to book from those tagged slots but others also 
	•	We should enable other properties on these tags as well. 
	•	Eg slot with Tag Type: WeekendOnly Tag Value: Yes should only be allowed to be booked on weekends.
	•	One slot can have more than one tags type, but only 1 tag type - tag value pair
	•	One employee can have more than one tags type, but only 1 tag type - tag value pair

Important Note: The solution should take care of bifurcation b/w 2 wheeler or 4 wheeler
Eg: if an employee is assigned a dedicated “employee” slot which only accommodates 2 wheeler, and if  the employee tries to book a 4 wheeler booking, he should be assigned 4 wheeler parking from hotslot or from his team slot (if he is a part of any team that has dedicated slots).

Auto Allocation:
I. Employee with dedicated “Employee” slot:
	•	If s/he has only one dedicated slot: When the employee tries to create the WFO booking the page should have the dedicated slot preselected (even before he clicks on “Add parking”)
	•	However if the employee wants to change vehicle type s/he can and get auto allocated from hotslots available, if no hotslots available (Show message: “No parking slots available for ## wheeler”)
	•	If s/he has more than one dedicated slots: 
	•	One 2-wheeler and One 4-Wheeler: The employee should have the 4 wheeler parking slot pre-selected by default when s/he opens the WFO card (even before he clicks on “Add parking”)
	•	However if the employee selects a 2 wheeler s/he should automatically get the 2 wheeler slots assigned without having to search for zone, level
	•	Multiple 4 wheeler or 2 wheeler: Not handled as of now
II. Employee is part of a “team” slots
	•	If the slots are available: 
	•	When the employee clicks on “Add parking” s/he should have the zone and level selected for respective available slots for that team
	•	Eg: if there are 2 slots assigned to Marketing team: one is zone A: level 2 and second on Zone B: Level 1, if employee A of the team comes to book he should get he should have Zone A, Level 2 preselected and later if emp tries to book he should have Zone B, Level 1 preselected
	•	 If the “Team” slots are unavailable: 
	•	If the “Team” slots are unavailable but hotslots available:
	•	The employee should have zone and level selected for the available hotslots and there should be a message displayed saying “All the slots for your team are booked, please try to select from other available slots”
	•	If neither team slots nor hot slots available:
	•	Show a message saying:Show message: “No parking slots available for ## wheeler”
	•	Same process should follow if the employee chooses to change vehicle type
III. Employee neither has a dedicated “Employee” slots nor “Team” slots:
	•	If hotslots available, the employee should have preselected zone and level for the hotslot available  (sequential)
	•	If hotslots unavailable, Show a message saying:Show message: “No parking slots available for ## wheeler”.
	•	Same process should follow if the employee chooses to change vehicle type
