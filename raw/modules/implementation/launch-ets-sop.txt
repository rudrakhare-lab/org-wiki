Launching WorkinSync on Existing MoveInSync Client Server  (Need to take signoff from Ujjwal before publishing this sheet to others)

Objective: This document outlines the step-by-step process for launching the WorkinSync (WIS) Workplace solution for existing MoveInSync employee transportation server. It is crucial that all steps are followed to ensure a smooth transition from the current system to the integrated WIS solution.

1. Pre-Launch Preparation
1.1 Decide on System Downtime
	•	Identify the date and time when the system will be offline for migration.(Preferably when there is very less usage)
	•	Ensure communication is sent to all stakeholders about the downtime. Input required: Downtime schedule and communication plan
	•	BCP : Have a backup plan in case something goes wrong. (Have manual trip sheets ready to run the transport operation)
1.2 Prepare Employee Data
	•	Gather and validate employee data from HR or internal records.
	•	If data is flowing through HRMS, Identify and tag any non-transport users (employees who do not have privilege to use transportation services) in the employee profile section. Example: User Profile with ID,  Transport user (Yes/No).
1.3 Operations Study Document
	•	Keep the operations study document ready with the following details:
	•	Shift timings
	•	Required features to be enabled
	•	Transportation cut-off times (Edit,cancel,No show,min&max shift duration)
	•	Desk booking cut-off times (Edit,cancel,min&max shift duration) Input required: Complete operations study document that was shared.


1.4 Prepare Floor Plan
	•	Ensure that the office floor plan is updated, indicating all desk numbers.
	•	Mark emergency exits,Entry, and other relevant sections before uploading to avoid back and forth uploads. Get a signoff on POC before uploading on PROD server.
1.5 Desk Allocation File
	•	Prepare the desk allocation file, mapping desks to employees. Input required: Desk allocation mapping (Allocation type, employee/team name, desk number).
1.6 Decide Check-In Mechanism
	•	Define the process employees will follow to check in to their desks (e.g., Digipass, Scan QR, Access card etc).
	•	Ensure this mechanism aligns with their policy (Especially while launching Geofence restriction). Input required: Check-in method to be used (App-based/RFID).
1.7 WIS Configuration Details
	•	Have all the WIS configuration details (e.g.,Features and relevant data, Parking/Meeting rooms/ cafeteria etc ) ready and agreed upon.
	•	Ensure all the relevant WIS reports are enabled. Input required: Finalized WIS features to be configured.
1.8 Backup Data (Ensure these informations is scrapped within 60 days from your laptops after the successful launch)
	•	Raise an SE ticket to get a configuration backup of the current system.
	•	Backup the following data:
	•	Employee data (including non-transport users)
	•	Vehicle data
	•	Driver data
	•	Transport booking data (If you are not doing a manual download of booking data, Ensure you validate the data shared by the tech team as there were multiple discrepancies in the past.)  Input required: SE ticket number for configuration backup.
1.9 Create a Test User (DO NOT SKIP THIS STEP)
	•	Create one test user and create a transport booking to test after the migration. 
	•	Ensure you complete the routing and assign a vehicle for this user.
	•	Ensure this booking reflect on the Mobile app with the OTP
	•	Take a screenshot of the transport booking on the mobile app.


2. Launch
2.1 Create Shifts 
	•	Set up shifts in the server, categorizing them into:
	•	Desk shifts (for non-transport users)
	•	transport shifts
	•	Raise SE ticker to ensure these are correctly configured on the Shift Categories Page on ETS Side.
2.2 Raise SE for Configuration Updates
	•	Submit an SE ticket to update the necessary configurations in the system. (Features and cutoffs)  Input required: SE ticket number for configuration updates.
2.3 Upload Floor Plan and Parking Plan etc.
	•	Raise an SE request to upload the floor plan and the parking plan  (if applicable) into the system. Input required: Cleaned up SVG and JSON floor/parking plan uploads.
2.4 Migrate Existing Transport Bookings (IMPORTANT)
	•	Migrate all existing transport bookings to WIS.
	•	Akshay/Anirban/Pranjal/Anoop can assist with the API for migration.
	•	The Bookings created for transport before the WIS migration will only appear on the mobile app only after the migration is completed


3. Testing
3.1 Verify Transport Booking
	•	Open the app and log in as a user with a transport booking that was created before the launch.
	•	Check if the existing bookings (pre-migration) are visible in the app.
	•	Validate if the OTP is appearing is same as the one that was reflected earlier
	•	Check the Routes page to ensure if the routing and booking count is not affected.
   3.2 If all the above steps are fine, Create a Fresh Booking
	•	Attempt to create a fresh transport booking for a future date via the mobile app along with the desk.
	•	Verify if the booking appears correctly on the WorkPlanner.

 4. Troubleshooting Guide
4.1 Booking Does Not Appear on Both Mobile App and WorkPlanner
	•	Issue: Migration is not successful.
	•	Action: Re-check the migration process and ensure all bookings were transferred correctly and appear on Work Planner. Try to Re-upload all the future bookings in the new format if the API doesn’t help migrating.
4.2 Booking Appears on WorkPlanner But Not on the Mobile App
	•	Issue: App display issue.
	•	Action:
	•	Uninstall and reinstall the latest version of the app.
	•	If the issue persists, consult the Mobile App Team for further resolution.
4.3 Booking Appears on Mobile App But Not on WorkPlanner
	•	Issue: WorkPlanner sync issue.
	•	Action: Contact the Tech Team to check the synchronization between the mobile app and WorkPlanner.
4.4 Booking Appears With a Different OTP
	•	Issue: Could lead to Serious Transport operational issues. Overlapping booking is created
	•	Action: Involve the Tech Team immediately to investigate the OTP mismatch. This could cause problems for users during Login, so prioritize fixing this. If nothing helps, remove all the booking and redo the booking.  Ensure the past bookings are removed from the ETS side as well before reuploading the bookings on the WIS side.
4.5 OTP on Mobile app is not accepted on driver device
	•	Issue: Improper trip-sheet display.
	•	Action: Ask the driver to clear caches on the mobile app or uninstall the app and reinstall. Talk to the Employee experience team and ask them to resolve the issue with an incorrect trip sheet.If nothing helps, remove all the booking and redo the booking.  Ensure the past bookings are removed from the ETS side as well before reuploading the bookings on the WIS side.

4.6 Inconsistent Desk Allocation After Migration
	•	Issue: Users see incorrect desk assignments or can not see any seat available.
	•	Action: Double-check the desk allocation file used during migration. Consult the client to ensure desk assignments are mapped correctly and updated on the system.It could also be possible all the seats allocated to their team are already reserved.
4.7 Users Unable to Access WIS Features Post-Launch
	•	Issue: Missing user profile or user logs in using an email or phone number that's not registered.
	•	Action: Ensure you check details employee use to logging into the system
4.8 Transport Bookings Not Migrated for Specific Users
	•	Issue: Partial migration issue.
	•	Action: Identify affected users and review the migration logs. If specific users' data was not transferred, involve Akshay/Anirban to use the migration API for those users. Ensure no booking data was lost in the process. (If this is time sensitive, create a fresh booking)
4.9 Desk Shift Not Appearing for Non-Transport Users
	•	Issue: Non-transport user shifts not configured properly or shift refresh issue
	•	Action: Recheck shift category configuration to ensure desk shifts are visible for non-transport users. Raise an SE ticket if  manual shift refresh is needed. Also check if the team is tagged to the right shift category under Manage teams section.
4.10  Non Transport Shift appears as Transport Shift or Vice versa.
	•	Issue: Incorrect Shift configuration.
	•	Action: Verify the shift configurations in the system and reassign transport users to the correct shift category. Consult the SE  for correct shift mapping.


4.11 Floor Plan Not Visible on WIS
	•	Issue: Floor plan not uploaded correctly.
	•	Action: Ensure the SE ticket to upload the floor plan is completed. Revalidate the plan on the platform and consult the Tech Team if it still fails to appear post SE Team confirmation. 
4.12 Incorrect Parking Plan After Migration
	•	Issue: Parking plan mismatches with actual parking assignments.
	•	Action: Verify that the correct parking plan was uploaded. If incorrect, raise a new SE ticket to re-upload the updated parking plan and ensure it reflects properly on the WIS platform.
4.13 Users Not Receiving Transport Booking Notifications
	•	Issue: Notification failure post-migration.
	•	Action: Verify the notification settings in the system and ensure they are enabled for all users. Work with the Mobile App Team to ensure push notifications are functioning correctly.
4.14 Bookings Showing Incorrect Time or Dates
	•	Issue: Time/date mismatch for transport bookings.
	•	Action: Investigate the source of the incorrect time or date, such as time zone settings or booking configurations. Work with the Tech Team to fix the issue. This happens only for the International clients 
4.15 Transport Availability Not Matching Operations Study
	•	Issue: Incorrect configuration of Shift in shift category page.
	•	Action: Cross-check the transport shift timings with the operational study document. Make necessary adjustments in the system or consult the SE Team for configuration updates.
4.16 Duplicate Bookings Showing on Mobile App
	•	Issue: Duplicate bookings for users. (Serious operational issue)
	•	Action: Identify the cause of duplicate entries in the migration logs. Have the Tech Team clean up the duplicate bookings from the system.

4.17 App Crashes During Booking Process
	•	Issue: App instability during transport booking.
	•	Action: Share the screen recording of the crash along with some basic details App version: OS: Android/IOS Affected Users details: Empid or Email Affect user credentials : If possible (If its a test profile)
 4.18 Missing Configuration
	•	Issue: Configuration backup not completed by SE team before migration.
	•	Action: Raise an Blocker SE ticket to back up the system's current configuration and prevent any potential operational issue. Ensure future testing does not proceed without this configuration completed.
4.19 Available Desk Booking Failing
	•	Issue: Issue with Cutoff configuration or tech issue
	•	Action: Ensure the desk booking workflow is correctly configured. Consult the Tech Team to troubleshoot and resolve any workflow issues in the system.
4.20 Transport Booking Cut-Off Time is Incorrect
	•	Issue: Transport booking cut-off time conflicts with WIS booking. 
	•	Action: Confirm that the transport booking cut-off times are in sync with the agreed WIS cutoff. Raise an SE ticket if there are discrepancies.
4.21 User Profile Data Incomplete After Migration
	•	Issue: Missing fields in user profiles post-migration.
	•	Action: Identify which fields are missing and ensure the employee data was backed up properly. Request the HR Team or Data Team to provide missing details, and update the profiles in the system. In case of HRMS integration in place
4.22 Users Facing Delays in Vehicle number Confirmation
	•	Issue: Vehicle is not assigned to the route by Admin.
	•	Action: Check the Routing page if the vehicle is assigned to that route. If Yes but the app still doesn't reflect the vehicle number, try to unassign and assign the vehicle on the routing page. >> Connect with the Emp Exp team if the issue continues.


4.23 Unable to Cancel Existing Transport Bookings
	•	Issue: Cutoff is passed.
	•	Action: Check if the issue is legit and within cutoff. 
4.24 Incorrect User Permissions for WIS Features
	•	Issue: Users unable to access specific WIS features due to permission errors in setting > Roles.
	•	Action: Review user role settings and ensure permissions are correctly assigned based on user profiles (Global admin/Front office/Employee/MSU/Team Manager etc). Involve the SE Team or PM to update user roles if needed.

Conclusion
Once all steps are successfully completed, WIS will be integrated with MoveInSync for desk and transport management. Ensure to track the system’s performance post-launch and troubleshoot any user issues promptly.
Note: All SE and TS tickets should be tracked, and system logs should be regularly monitored throughout the process.
